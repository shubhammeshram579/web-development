🔄 What is a Transaction in MongoDB?

A transaction means:

A group of database operations that must either all succeed or all fail.

This is called ACID:

Atomic → All or nothing

Consistent → Data always valid

Isolated → Safe from other operations

Durable → Once saved, it stays

🏦 Real-Time Example (Bank Transfer)

If Shubham sends ₹5000 to Rohit:

Deduct ₹5000 from Shubham’s account

Add ₹5000 to Rohit’s account

If step 2 fails, step 1 must rollback.
Otherwise money is lost 😨

This is why we need transactions.

🛒 Real-Time Example (E-commerce Order)

When placing an order:

Create order

Reduce product stock

Deduct wallet balance

If any step fails → rollback everything.

✅ When Should You Use MongoDB Transactions?

Use transactions when:

Payment systems

Wallet systems

Order + inventory updates

Booking systems

Multiple collection updates

Do NOT use for:

Simple read operations

Single document updates (MongoDB already atomic per document)

⚙️ MongoDB Transaction Requirements

MongoDB transactions work only with:

Replica Set OR

MongoDB Atlas

Not supported on standalone MongoDB.

🔥 Express.js Transaction Example (Place Order)
const mongoose = require("mongoose");
const User = require("./models/User");
const Order = require("./models/Order");

app.post("/api/place-order", async (req, res) => {
  const session = await mongoose.startSession();

  try {
    session.startTransaction();

    const { userId, product, amount } = req.body;

    // 1️⃣ Check user balance
    const user = await User.findById(userId).session(session);

    if (!user || user.balance < amount) {
      throw new Error("Insufficient balance");
    }

    // 2️⃣ Deduct balance
    user.balance -= amount;
    await user.save({ session });

    // 3️⃣ Create order
    const order = await Order.create(
      [{ userId, product, amount, status: "PLACED" }],
      { session }
    );

    // 4️⃣ Commit transaction
    await session.commitTransaction();
    session.endSession();

    res.json({
      success: true,
      message: "Order placed successfully",
      order
    });

  } catch (error) {
    // ❌ Rollback
    await session.abortTransaction();
    session.endSession();

    res.status(500).json({
      success: false,
      message: error.message
    });
  }
});

💡 How Transaction Flow Works
Start Session
   ↓
Start Transaction
   ↓
Run all DB operations
   ↓
If success → Commit
If error → Abort (Rollback)
   ↓
End Session

🚀 Advanced Example (Money Transfer)
app.post("/api/transfer", async (req, res) => {
  const session = await mongoose.startSession();

  try {
    session.startTransaction();

    const { fromUserId, toUserId, amount } = req.body;

    const fromUser = await User.findById(fromUserId).session(session);
    const toUser = await User.findById(toUserId).session(session);

    if (fromUser.balance < amount) {
      throw new Error("Insufficient balance");
    }

    fromUser.balance -= amount;
    toUser.balance += amount;

    await fromUser.save({ session });
    await toUser.save({ session });

    await session.commitTransaction();
    session.endSession();

    res.json({ success: true, message: "Transfer successful" });

  } catch (error) {
    await session.abortTransaction();
    session.endSession();

    res.status(500).json({ success: false, message: error.message });
  }
});

🏆 Best Practices

✅ Keep transactions short
✅ Use indexes for transaction queries
✅ Do not perform heavy reads inside transaction
✅ Always handle rollback
✅ Use try-catch properly
